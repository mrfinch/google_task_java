Command Line Tasks API Sample Instructions

A starter application has been generated for you.

  1. Install Java Platform (JDK 6 or higher).
  2. Install Maven (3.1.0 or higher). You may need to set your JAVA_HOME.
  3. Download the starter application and unzip it.
  4. Download the client secrets file and overwrite the src/main/resources/client_secrets.json file that was included in the application.
  5. Open a terminal, change to the tasks-cmd-line-sample directory you just unzipped, and execute: 
    mvn compile
mvn -q exec:java
    This will compile and run your starter application in the command line.

Congratulations, you are up and running! At this point you will want to add calls into the Tasks API to the /src/main/java/com/google/api/services/samples/tasks/cmdline/TasksSample.java file. Please read the file carefully, it contains detailed information in the comments.

To enable logging of HTTP requests and responses (highly recommended when debugging), please take a look at the logging.properties file.

For more information on the Tasks API Java library surface you can visit:

https://develpers.google.com/resources/api-libraries/documentation/tasks/v1/java/latest/

Also check out the Java Client Library documentation, and get more information on the Tasks API at:

https://developers.google.com/google-apps/tasks/firstapp

Setup Project in Eclipse

Prerequisites: install Eclipse, and the Maven plugin.

  • Setup Eclipse Preferences
    • Window > Preferences... (or on Mac, Eclipse > Preferences...)
    • Select Maven
      • check on "Download Artifact Sources"
      • check on "Download Artifact JavaDoc"
  • Import tasks-cmd-line-sample project
    • File > Import...
    • Select "General > Existing Project into Workspace" and click "Next"
    • Click "Browse" next to "Select root directory", find tasks-cmdline-sample and click "Next"
    • Click "Finish"
  • Run
    • Right-click on project tasks-cmd-line-sample
    • Run As > Java Application
    • If asked, type "TasksSample" and click OK
